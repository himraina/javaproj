OSSearch™ v1.1.1 Installation Instructions:

  1. Set default values in 'src/com/nebulais/ossearch/OSSearchConstants.java'

    • ADMIN_EMAIL - The administrator's e-mail address
    • ADMIN_NAME - The administrator's name (i.e., the from field of the e-mail)
    • SERVLET_DIR - Servlet Directory (from root of the file system)
    • SERVLET_URL - The url to the servlet's directory (include http://)

  2. Build the package: From the 'ossearch_v1.1.1' directory execute 'ant' [or build without Ant] 
    > cd /ossearch/install/path/ossearch_v1.1.1
> ant
  3. Edit values in 'data/ossearch/ossearch.cfg'
    1. #root_url http://www.yoururl.com/
      Url to the top directory in the search. Must start with http://.

    2. #root_path /your/htdocs/directory/
      File path from the root of the file system to the top directory in the search.

    3. #recursive true
      true or false? Should the directory structure be recursively searched?

    4. #extensions .html .htm
      Whitespace separated list of file extensions to allow in the search.

    5. #exclude_dirs
      Whitespace separated list of directories to exclude from the search (relative to #root_path).

    6. #exclude_files
      Whitespace separated list of files to exclude from the search (relative to #root_path).

    7. #end
      Marks the end of the configuration file.

  4. Create a template file and save it to ossearch_1.html. Take any html file you like and the keywords below. The keywords will be filled-in as described below and returned to the user as the result of the Servlet.
    • %results% — Inserts the results from the search. An order list of up to 15 links.
    • %keywords% — Inserts a space separated list of the keywords which caused the results.
    • %servlet_url% — Url to the servlets directory including 'http://'. To get the url to your search engine servlet in the template, add '%servlet_url%/OSSearchServlet' to your template. This will expand to 'http://www.yoururl.com/your/servlet/directory/OSSearchServlet'

    • Note: A default template is supplied.
    • Note: Try to avoid using "%" signs in your templates.
    • Note: Make sure 'other' has permission in the #root_path directory (755)

  5. Make sure the JSDK1.2 or higher is in your servlet environment. If not, unjar jsdk.jar in your SERVLET_DIR.

  6. Upload package to your server using Ant [or install without Ant]

    1. Set ftp property values in 'build.xml' in the top level directory of OSSearch™
      • SERVER_NAME — Url or IP address of the web server to deploy OSSearch™ on.
      • USERNAME — Log-in name for the web server.
      • PASSWORD — Password for the given username on the web server. For security reasons, it's recommend that this value is removed after the upload is complete.
      • SERVLET_DIR — Path to the servlet directory to deploy OSSearch™ in (relative to the username's home directory).

    2. From the 'ossearch_v1.1.1' directory execute 'ant install' 
      > cd /ossearch/install/path/ossearch_v1.1.1
> ant install
      Note: Other Ant ftp targets include, 'uninstall' and 'reinstall.' Uninstall removes all OSSearch™ files from the web server. Reinstall only uploads OSSearch™ files which are newer than those already on the web server.

  7. Integrate into your web site. Cut and Paste the following form into your web page(s).

    Note: Replace www.yoururl.com with your domain name or IP Address.
