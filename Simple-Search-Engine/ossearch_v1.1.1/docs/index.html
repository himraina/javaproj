OSSearch™ v1.1.1 Documentation:


Synopsis

OSSearch™ is a web site search engine Java Servlet. OSSearch™ searches each web page for the give keywords and ranks them based on a number of criteria (See Details). The search results can be easily integrated with your web page layout (See Details).

Installation

This package has been tested with Tomcat and JServ. And, has been tested on both Windows and Linux platforms. A basic knowledge of HTML, Java and Servlet programming is needed. However, installation should be quite simple. This package requires Jsdk1.2 or higher. The necessary jar file is included in the OSSearch™ package. Please, do not individually redistribute this jar file. Obtain copies of this JavaSoft package for redistribution by visiting, http://www.javasoft.com/. Redistributing jsdk.jar as part of package descendant from OSSearch™ is permitted.

Details

  • Since OSSearch™ parses each file per search, it should only be used in medium sized web sites; hundred or so pages depending on speed of Java interpreter (JSDK 1.3 with Tomcat speeds up the search considerably)
  • Search ranking is based on the following:
    • Number of keyword matches in document
    • Proximity of keyword matches (i.e., phrase matching)
    • Bonus for all keywords matching when multiple keywords are given
    • Bonus points for keyword matches in title and meta tags (keywords & descriptions)
  • The following preferences can be set:
    • File extensions to search
    • Whether to recursively search the directory structure.
    • Files to exclude
    • Directories to exclude
  • Html file templates are used to construct the search result page (post) and blank search page (get). Keywords for the following values can be inserted in to a html page (on post) of the developers choose.
    • results - Results of the search
    • keywords - Keywords of the search
  • Commonly used pronouns and other short words are excluded from the search.
